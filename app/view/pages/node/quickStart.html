快速安装使用

安装

安装流程可参考这篇文章：node环境（nvm+node+npm+环境配置）安装

日常开发中电脑上可能装有多个版本的Node.js，可以使用nvm来对Node.js进行版本切换。nvm下载地址

Node.js有长期支持版（LTS）和最新特性版之分，建议生产环境使用长期支持版。Node.js官网下载地址

安装Node.js的时候会自动安装好npm的，npm是Node.js的包管理工具。npm官网地址

通过npm安装模块时都是去国外的镜像下载的，有的时候由于网络原因会导致安装模块失败，可以使用淘宝提供的镜像进行安装。淘宝 NPM 镜像

使用

1. 检查是否都已安装成功

在命令行中分别输入以下命令以检查安装的版本。

nvm
node -v
npm -v

2. nvm常用命令

若想更新nvm，可以通过Git将新版nvm目录从GitHub上拉取到本地，相当于重装了一次。

命令 说明
nvm 输出版本号及使用说明
nvm list [available] 查看已安装了的Node.js版本，加上参数available，查看目前所有可安装的Node.js版本
nvm install <version> 安装指定的Node.js版本，如：nvm install v7.9.0
nvm uninstall <version> 卸载指定的Node.js版本
nvm use [version] 切换为指定版本的Node.js进行使用，如：nvm use v7.9.0
nvm node_mirror [url] 设置Node.js的下载地址，默认为 https://nodejs.org/dist/

3. npm常用命令

升级 npm 可使用以下命令：

npm install npm -g
命令 说明
npm -v 返回当前npm的版本
npm init 在项目中引导创建一个package.json文件
npm install xx@1.1.1 安装模块，不加版本号则安装最新版。-g表示全局安装，-S表示写入dependencies中，-D表示写入devDependencies中，-O表示写入optionalDependencies中
npm uninstall xx 卸载安装的模块,-g表示卸载全局的
npm update xx 升级安装的模块,-g表示升级全局的
npm list xx 查看当前项目中安装的模块信息，-g表示查看全局安装的。不指定xx则查看所有的
npm outdated 列出已经过时的模块，以便及时升级
npm config set 管理npm的配置，如registry='https://registry.npm.taobao.org/'，原官方地址为：'http://registry.npmjs.org'
npm config list 显示配置信息
npm cache clean 清除npm本地缓存
npm run xx npm run 会创建一个新的shell，执行指定的命令（xx是定义在package.json文件的scripts字段中的），因此它不一定是 Node 脚本，任何可执行文件都可以写在里面。不指定xx，则列出所有scripts中的脚本命令及内容。
npm start npm run start 的简写，若没有设置start字段则执行node server.js
npm who am i 查看当前npm账号
npm adduser 按提示注册npm官网账号
npm login 按提示登陆npm官网账号
npm publish 发布npm包到npm官网

4. node常用命令

命令 说明
node 回车 进入node交互模式，可直接执行js代码并输出结果
node xxx.js 执行某个js文件
